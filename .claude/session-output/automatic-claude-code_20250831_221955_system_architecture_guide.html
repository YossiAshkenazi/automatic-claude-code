üîÑ Communication Flow

Data Flow Between Components

üéØ From External Project to Agents

1
User Command: User runs command from external project

2

  • Manager Agent: High-level planning (Opus model)
  • Worker Agent: Task execution (Sonnet model)
3

  • AGENT_COORDINATION
  • MANAGER_TASK_ASSIGNMENT
  • WORKER_PROGRESS_UPDATE
  • MANAGER_QUALITY_CHECK

üì° From Agents to Monitoring System

1
Event Emission: Agent Coordinator calls emitCoordinationEvent()

2
HTTP POST Request: Monitoring data sent to server
POST http://localhost:4001/api/monitoring
{ "agentType": "manager", "messageType": "coordination_event", "message": "MANAGER_TASK_ASSIGNMENT", "metadata": { ... }, "sessionInfo": { "task": "create auth system", "workDir": "/path/to/project" } }
3
Database Storage: Server stores session and message data
  • Session creation/update in database
  • Agent message storage
  • Analytics metrics collection
4
WebSocket Broadcast: Real-time data sent to connected clients
broadcast({ type: 'agent_message', data: agentMessage });

üöÄ External Project Integration

‚úÖ How External Projects Connect

Any project can use the automatic-claude-code system by running commands from their directory:

Step-by-Step Integration Process

1
Project Setup: Navigate to your project directory
cd C:\Users\Dev\orthodox-synagogue-auction-platform
2
Run Command: Execute automatic-claude-code with relative path
node "../automatic-claude-code/dist/index.js" run "your task here" --dual-agent -i 5 -v
3
Monitoring Starts: System automatically starts monitoring server if enabled
  • Checks if monitoring server is running on port 4001
  • Starts server if not running
  • Provides UI URL for real-time monitoring
4
Agent Execution: Dual-agent system processes the task
  • Manager analyzes task and creates work plan
  • Worker executes specific development tasks
  • All coordination events are sent to monitoring system
5
Real-time Monitoring: View progress in dashboard
UI Dashboard: http://localhost:6011 API Health: http://localhost:4001/api/health Session Data: http://localhost:4001/api/sessions

Configuration Options
